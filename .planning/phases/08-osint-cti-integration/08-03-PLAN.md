# Plan 08-03: Source Reputation & Multi-Source Consensus

**Goal:** Implement dynamic reputation system for OSINT sources and extend ConsensusEngine to handle OSINT multi-source consensus with conflict detection.

## Requirements

- OSINT-03: Enhance Graph Investigator with OSINT integration (confidence scoring, cross-referencing)
- CTI-02: Multi-source verification and cross-referencing

## Context

Phase 8 requires aggregating findings from 15+ OSINT sources and determining threat status through consensus. Rather than trusting any single source, the system should track source accuracy over time and dynamically adjust weights. Conflicting findings must be preserved with reasoning, not forced into consensus.

**Current State:**
- ConsensusEngine exists for multi-source validation but Vision/Security-focused
- No source reputation tracking
- No OSINT-specific conflict detection (threat vs safe)
- Static weights, no dynamic adjustment

**Target State:**
- SourceReputation tracks accuracy, false positives, false negatives
- ReputationManager calculates weighted consensus scores
- ConsensusEngine extended for OSINT findings
- Threat vs safe conflict detection with preservation
- 3+ source agreement threshold (or 2 high-trust sources)

**Locked Decisions (08-CONTEXT.md):**
> Dynamic Reputation System: Sources build trust/reputation over time through accurate threat predictions. Self-correcting system: sources that consistently get predictions right gain weight.
>
> Conflict Resolution: Preserve contradictions with detailed reasoning. Show conflicting results transparently.

   - confidence: float (0.0-1.0)

   - source: str
   - total_predictions: int (default 0)
   - correct_predictions: int (default 0)
   - false_positives: int (default 0) - predicted malicious, actually safe
   - false_negatives: int (default 0) - predicted safe, actually malicious
   - recent_predictions: List[SourcePrediction] (default factory)
   - accuracy_score property: correct/total (0.5 if total=0)
   - recent_accuracy(days=30) property: accuracy from recent predictions
   - false_negative_rate property: FN/total (0.0 if total=0)
   - false_positive_rate property: FP/total (0.0 if total=0)
   - weighted_reputation property: weighted score (accuracy 60% + recent factor 20% + FN penalty 20%)

4. ReputationManager class:
   - __init__(): creates sources dict, initializes core sources (dns, whois, ssl, urlvoid, abuseipdb)
   - _get_or_create(source_name: str) -> SourceReputation
   - record_prediction(source: str, verdict: VerdictType, confidence: float=0.8)
   - record_actual(source: str, prediction_index: int, actual: VerdictType)
   - calculate_consensus_weight(source_name: str) -> float: returns weight for consensus calculation
   - get_confidence_thresholds() -> dict[str, float]: returns per-source min confidence thresholds

weighted_reputation formula: (base_accuracy * 0.6) + (recent_factor * 0.2) + (fn_penalty * 0.2)
where recent_factor = min(1.0, recent_accuracy / (base_accuracy + 0.01))
and fn_penalty = 1.0 - min(1.0, false_negative_rate * 3)

Extend ConsensusEngine for OSINT consensus in veritas/quality/consensus_engine.py

Add OSINT consensus methods to ConsensusEngine class:

1. compute_osint_consensus(results: dict[str, OSINTResult], min_sources=3, exception_high_trust=True) -> dict:
   - Filter to valid results where status=SUCCESS
   - Call _osint_result_to_verdict() for each result to get verdict string
   - Count agreement for each verdict (malicious, safe, suspicious, unknown)
   - Find dominant verdict with most sources
   - Check for conflict: has both malicious and safe sources
   - Call _determine_osint_status() for consensus status
   - Call _generate_osint_reasoning() for human-readable explanation
   - Return dict with: consensus_status, verdict, agreement_count, total_sources, conflicting_sources, has_conflict, reasoning

2. _osint_result_to_verdict(result: OSINTResult) -> str:
   - For THREAT_INTEL: malicious if abuse_confidence>50 or reports>5, suspicious if >20 or >2, else safe
   - For REPUTATION: malicious if detections>3 or risk==high, suspicious if detections>0 or risk in [low,medium], else safe
   - For WHOIS/SSL: suspicious if age_days<30 or !is_valid, else safe
   - Default: unknown

3. _determine_osint_status(agreement_count, total_sources, has_conflict, min_sources=3, exception_high_trust=True) -> str:
   - If has_conflict: return "conflicted"
   - If agreement_count >= min_sources: return "confirmed"
   - If exception_high_trust and agreement_count >= 2: return "confirmed"
   - Check ratios to return "likely", "possible", or "insufficient"

4. _generate_osint_reasoning(status, verdict, agreement, total, has_conflict, malicious, safe) -> str:
   - Generate human-readable explanation based on status and counts
   - For conflicts: list conflicting sources explicitly
   - For confirmed/likely/possible/insufficient: provide clear reasoning

Must preserve conflicts with detailed reasoning.
