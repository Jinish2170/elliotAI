# Plan 08-01: Core OSINT Intelligence Sources

**Goal:** Establish foundational OSINT infrastructure with DNS, WHOIS, and SSL sources wrapped for async execution and cached with source-specific TTLs.

## Requirements

- OSINT-01: Implement 6+ OSINT intelligence sources (DNS, WHOIS, SSL core sources plus extensible framework)

## Context

**Current State:**
- GraphInvestigator has basic WHOIS, DNS, SSL verification but synchronously implemented
- No centralized OSINT source architecture
- No OSINT result caching
- No source-specific TTL management

**Target State:**
- Modular OSINT source architecture with standardized interface
- Async wrappers for DNS, WHOIS, SSL to prevent blocking
- SQLite persistence with source-specific TTLs (WHOIS 7d, SSL 30d, DNS 24h)
- Reusable OSINTResult dataclass for source outputs

**Locked Decision (08-CONTEXT.md):**
> API Tiering Strategy: Core OSINT domains work without API keys (DNS records, WHOIS, SSL certificate validation).

<description>
Implement SSL certificate validation in veritas/osint/sources/ssl_verify.py

Create SSLSource class with:
1. async query(hostname: str, port: int = 443) -> OSINTResult:
   - Use asyncio.run_in_executor() for blocking SSL operations
   - Create SSL context with ssl.create_default_context(ssl.Purpose.SERVER_AUTH)
   - Enable hostname verification and certificate validation
   - Connect to hostname:port, retrieve certificate via getpeercert()
   - Parse certificate fields:
     * notBefore, notAfter (parse with datetime.strptime)
     * issuer, subject (extract from lists)
     * subjectAltName (extract list of values)
     * serialNumber, version
   - Calculate days_until_expiry = (not_valid_after - datetime.utcnow()).days
   - Extract issuer_org from issuer dict organizationName
   - Check if_valid = days_until_expiry > 0
   - Check is_expiring_soon = days_until_expiry < 30
   - Return OSINTResult with:
     * data: {hostname, subject, issuer, issued_at, expires_at, days_until_expiry, subject_alt_names, version, serial_number, is_valid, is_expiring_soon, issuer_org}
     * confidence: 0.9 if days_until_expiry > 30, 0.5 otherwise
   - Handle connection/certificate errors

Must use proper SSL context with verification enabled.

Create OSINT cache with source-specific TTLs in veritas/osint/cache.py

Create OSINTCache class with:
1. SOURCE_TTLS dict with timedelta values:
   - "dns": timedelta(hours=24)
   - "whois": timedelta(days=7)
   - "ssl": timedelta(days=30)
   - "abuseipdb": timedelta(hours=12)
   - "urlvoid": timedelta(hours=24)
   - "social": timedelta(hours=24)

2. static method generate_cache_key(source: str, category: OSINTCategory, query_type: str, **query_params) -> str:
   - Normalize all params to lowercase strings
   - Create JSON dict sorted alphabetically
   - Return MD5 hash hex string

3. static method get_cached_result(session, cache_key: str) -> Optional[Dict]:
   - Query OSINTCache model for cache_key with expires_at > utcnow()
   - Return dict with source, result, confidence_score, cached_at, expires_at, from_cache=True
   - Return None if not found or expired

4. static method cache_result(session, cache_key: str, result: OSINTResult) -> None:
   - Get TTL from SOURCE_TTLS dict (default to 24h if source not found)
   - Calculate expires_at = utcnow() + ttl
   - Delete any existing record with same cache_key
   - Create new OSINTCache model record
   - Commit to database

Must import OSINTCache model from veritas.db.models (avoid circular import).
