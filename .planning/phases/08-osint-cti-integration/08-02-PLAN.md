# Plan 08-02: OSINT Orchestrator with Smart Fallback

**Goal:** Create intelligent CTI/OSINT orchestrator that manages API resources, tracks rate limits, and gracefully falls back to alternative sources when queries fail.

## Requirements

- OSINT-01: Additional OSINT sources (URLVoid, AbuseIPDB) - progress toward 6+ sources
- CTI-02: Multi-source verification and cross-referencing orchestrator

## Context

Phase 8 requires coordinating 15+ OSINT sources with varying rate limits and availability. The orchestrator provides centralized management: query routing, rate limit tracking, circuit breaking to prevent API throttling, and intelligent fallback to alternative sources when primary sources fail.

**Current State:**
- No central OSINT management
- No rate limiting or quota tracking
- Sources called independently without coordination
- No alternative source fallback

**Target State:**
- OSINTOrchestrator auto-discovers available sources
- Per-source rate limit tracking (requests per minute/hour)
- Circuit breaker pattern prevents API throttling
- Smart fallback tries alternative sources in same category
- Graceful degradation continues without problematic sources

**Locked Decisions (08-CONTEXT.md):**
> Create dedicated orchestrator class specialized for CTI and OSINT management. Implements smart fallback: when API fails, intelligently try other sources.

<description>
Add query_with_retry and _try_alternatives methods to OSINTOrchestrator

Add two methods to OSINTOrchestrator class:

1. async query_with_retry(source_name, method: str, *args, max_retries=2, **kwargs) -> Optional[OSINTResult]:
   - Get source and config, return None if missing or disabled
   - Check circuit breaker - return None if open for source
   - Check rate limits via rate_limiter.can_request() - return None if limited
   - Loop up to max_retries+1 attempts:
     * Call method on source via getattr(source, method)(*args, **kwargs)
     * On success: record_request(), return result
     * On TimeoutError: log warning, continue to next attempt
     * On other Exception: log warning, continue to next attempt
     * On final failure: record_failure() to open circuit breaker
   - If all retries fail: call _try_alternatives() and return None

2. async _try_alternatives(failed_source, method: str, *args, **kwargs):
   - Get alternative sources for category (use OSINTCategory.THREAT_INTEL as default)
   - For each of first 2 alternatives:
     * Call query_with_retry() for alternative source
     * If returns result, log success and return

Must handle method resolution and proper async/await patterns.
</description>
