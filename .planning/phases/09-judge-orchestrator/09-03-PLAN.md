# Plan 09-03: Real-time Progress Streaming

**Goal:** Implement real-time progress streaming with progressive screenshot delivery, finding notifications, and user engagement pacing to maintain "always something happening" experience during long audits.

## Requirements

- PROG-01: Progressive screenshot streaming with immediate capture, thumbnail delivery, live scroll visualization, connection health
- PROG-02: Real-time pattern/discovery notifications with immediate finding send, live feed, color-coded alerts, incremental confidence
- PROG-03: User engagement pacing with 5-10s activity signals, agent indicators, progress bars, countdown timers, interesting highlights

## Context

**Current State:**
- Progress events emitted via ##PROGRESS: stdout markers for IPC communication
- No WebSocket streaming to frontend
- No rate limiting on progress events
- Screenshots batched at end of audit
- No estimated completion time countdown

**Target State:**
- ProgressEmitter with token-bucket rate limiting (max 5 events/sec)
- Progressive screenshot streaming with thumbnail compression
- Batched finding notifications (5 findings per event)
- Agent status events with state tracking
- CompletionTimeEstimator with countdown timer
- User engagement pacing maintaining 5-10s activity signals
- Color-coded severity alerts (red=CRITICAL, orange=HIGH, yellow=MEDIUM, green=LOW/positive)

**Locked Decision (09-RESEARCH.md):**
> Use token-bucket rate limiting for WebSocket event throttling (5 events/sec).

<description>
Create token-bucket rate limiter in veritas/core/progress/rate_limiter.py

1. Create RateLimiterConfig dataclass with:
   - max_rate: float = 5.0 (events per second)
   - burst: int = 10 (max burst capacity)

2. Create Event dataclass with:
   - priority: int (0=highest)
   - data: dict (event payload)
   - timestamp: float

3. Create TokenBucketRateLimiter class with:
   - __init__(config=None):
     * Initialize bucket with burst tokens
     * Set max_queue_size=100, dropped_count=0
     * Initialize event_queue: deque[Event](maxlen=max_queue_size)
   - _refill_tokens():
     * Calculate elapsed time since last_refill
     * Refill: tokens += elapsed * max_rate
     * Cap at burst capacity
   - async acquire(event_data: dict, priority: int = 0) -> bool:
     * _refill_tokens()
     * If tokens >= 1.0: consume 1 token, return True (emit immediately)
     * Else: queue event with priority, return False (rate limited)
     * If queue full: drop lowest-priority event to make room (if incoming priority <= lowest)
   - async get_queued_event() -> Optional[dict]:
     * Wait 100ms for token refill
     * Refill tokens
     * If queue has events AND tokens >= 1.0: pop event, consume token, return data
   - get_stats() -> dict:
     * Return tokens_remaining, tokens_capacity, queue_size, max_queue_size, dropped_events, max_rate

Token refill uses asynchronous lock for thread safety.
</description>

3. Create veritas/core/progress/emitter.py with ProgressEmitter class:
   - __init__(websocket=None, rate_limiter=None):
     * Initialize rate_limiter or create TokenBucketRateLimiter()
     * Set findings_batch_size=5, sequence_number=0
     * Initialize findings_buffer=[]
   - async emit_event(event_type, priority=EventPriority.MEDIUM, **kwargs):
     * Increment sequence_number
     * Build event dict: type, priority, seq, timestamp, **kwargs
     * Call rate_limit.acquire() -> allowed
     * If allowed and websocket: _send_event(event)
     * Else: log "Event rate-limited"
   - async emit_screenshot(screenshot_data, label, phase, thumbnail=True):
     * If thumbnail: resize to 200x150 via PIL, save as JPEG quality 70, base64 encode
     * Else: base64 encode full screenshot
     * Call emit_event(type="screenshot", priority=MEDIUM) with phase, label, image, size, is_thumbnail
     * Uses base64 and PIL.Image for image processing
   - async emit_finding(category, severity, message, phase, confidence=None):
     * Build finding dict: category, severity, message, phase, confidence (optional)
     * Append to findings_buffer
     * If buffer >= findings_batch_size: _flush_findings_buffer()
   - async _flush_findings_buffer():
     * Emit findings_batch event with all buffered findings
     * Clear findings_buffer
   - async emit_progress(phase, step, pct, detail=""):
     * Call emit_event(type="progress", priority=LOW) with phase, step, pct, detail
   - async emit_agent_status(agent, status, detail=""):
     * Call emit_event(type="agent_status", priority=HIGH if status=="failed" else MEDIUM) with agent, status, detail
   - async emit_error(error_type, message, phase, recoverable=False):
     * Call emit_event(type="error", priority=CRITICAL if not recoverable else HIGH) with error_type, message, phase, recoverable
   - async flush():
     * Call _flush_findings_buffer() to emit any buffered findings
   - async _send_event(event):
     * If websocket: await websocket.send_json(event)
     * Else: log warning "No WebSocket connection"

Thumbnail generation uses PIL thumbnail() method with 200x150 size.

<description>
Create completion time estimator in veritas/core/progress/estimator.py

1. Create AgentStatus enum: PENDING, RUNNING, COMPLETED, FAILED, SKIPPED

2. Create AgentExecution dataclass with:
   - agent: str (agent name)
   - site_type: str (site type classification)
   - start_time: float (timestamp)
   - end_time: float = 0.0
   - duration_ms: int = 0
   - status: AgentStatus = AgentStatus.PENDING

3. Create CompletionTimeEstimator class with:
   - __init__(ema_alpha=0.2, max_history=100):
     * Initialize EMA smoothing factor and history limit
     * Create _executions: list[AgentExecution]
     * Create _ema_history: dict[str, float] (site_type:agent -> EMA time)
   - start_agent(agent, site_type) -> AgentExecution:
     * Create execution, append to _executions, return
   - complete_agent(execution, success=True):
     * Set end_time, duration_ms, status (COMPLETED/FAILED)
     * Call _update_ema(execution)
   - _update_ema(execution):
     * key = f"{site_type}:{agent}"
     * ema = _ema_history.get(key)
     * new_ema = ema * (1 - alpha) + duration_ms * alpha (exponential moving average)
     * Store in _ema_history[key]
   - estimate_remaining(site_type, remaining_agents) -> int:
     * For each agent in remaining_agents:
       - key = f"{site_type}:{agent}"
       - ema = _ema_history.get(key)
       - If ema: total_ms += ema
       - Else: use hardcoded defaults (scout=20000, vision=30000, security=15000, graph=10000, judge=10000, osint=25000)
     * Return total_ms // 1000 (seconds)
   - get_eta() -> Optional[float]:
     * Get running agent and remaining agents
     * Call estimate_remaining()
     * Return estimated seconds remaining or None if no running agent

EMA formula: new_ema = old_ema * (1 - alpha) + new_value * alpha
</description>
