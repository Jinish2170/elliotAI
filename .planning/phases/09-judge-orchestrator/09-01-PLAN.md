# Plan 09-01: Judge Dual-Tier System

**Goal:** Transform Judge Agent from single-trust-score system to dual-tier verdict system with technical (CWE/CVSS) and non-technical (plain English) tiers, powered by 11 site-type-specific scoring strategies.

## Requirements

- JUDGE-01: Design dual-tier verdict data classes (VerdictTechnical with CWE/CVSS/IOCs, VerdictNonTechnical with plain English)
- JUDGE-02: Implement 11 site-type-specific scoring strategies using Strategy Pattern (GoF 1994)
- JUDGE-03: Build Judge Agent with dual-tier generation, single shared trust score, strategy pattern integration

## Context

**Current State:**
- JudgeAgent uses single TrustScoreResult (0-100 score with RiskLevel)
- trust_weights.py provides SignalWeights for 6 signals (visual, structural, temporal, graph, meta, security)
- OverrideRules apply hard-stop scoring logic
- No site-type-specific scoring adjustments
- No technical tier (CWE/CVSS) for security professionals

**Target State:**
- DualVerdict class combining technical and non-technical tiers
- 11 site-type strategies (Ecommerce, Financial, SaaS, Portfolio, News/Blog, Social, Education, Healthcare, Government, Gaming, Darknet)
- Strategy Pattern enabling runtime strategy switching
- CWE/CVSS integration for technical forensic reporting
- Backward compatible (existing code continues to work)

**Locked Decision (09-RESEARCH.md):**
> Use Strategy Pattern with abstract base class for 11 site-type scoring strategies.
