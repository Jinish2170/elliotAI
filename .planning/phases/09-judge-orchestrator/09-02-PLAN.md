# Plan 09-02: Smart Orchestrator

**Goal:** Transform orchestrator from fixed-timeout execution to adaptive orchestration with complexity-based timeouts, circuit breaker protection, and graceful degradation ensuring partial results even when agents fail.

## Requirements

- ORCH-01: Advanced time management with adaptive timeout strategies, parallel execution optimization, estimated completion time
- ORCH-02: Comprehensive error handling with circuit breaker, automatic fallback, graceful degradation ("show must go on")
- ORCH-03: Complexity-aware orchestration with threshold detection, analysis simplification, high-value prioritization

## Context

**Current State:**
- orchestrator.py uses fixed timeouts for each agent
- No circuit breaker protection for cascading failures
- No graceful degradation (agent failure = audit failure)
- No complexity-based timeout adjustment
- No estimated completion time calculation

**Target State:**
- TimeoutManager with adaptive timeouts based on ComplexityMetrics
- CircuitBreaker pattern with CLOSED/OPEN/HALF_OPEN states and exponential backoff
- FallbackManager with DegradedResult ensuring "show must go on"
- Orchestrator integration with adaptive timeouts and circuit breakers
- ComplexityAnalyzer collecting DOM depth, script count, lazy-load detection
- Historical learning for execution time estimation

**Locked Decision (09-RESEARCH.md):**
> Use circuit breaker pattern with exponential backoff for graceful degradation.

Create CircuitBreaker with state machine in veritas/core/circuit_breaker.py

1. Create CircuitState enum: CLOSED, OPEN, HALF_OPEN

2. Create CircuitBreakerConfig dataclass with:
   - failure_threshold: int = 3 (failures before tripping)
   - timeout_ms: int = 30000 (time to stay open before half-open)
   - half_open_max_calls: int = 1 (test calls in half-open)
   - success_threshold: int = 1 (successes to close circuit)

3. Create ResultWithFallback dataclass with:
   - value: Any (result data)
   - is_fallback: bool (primary vs fallback indicator)
   - fallback_reason: str (reason for fallback)
   - primary_error: Optional[str] (error from primary function)

4. Create CircuitBreaker class with methods:
   - __init__(name, config=None)
   - async call(primary_fn: Callable, fallback_fn: Optional[Callable] = None) -> ResultWithFallback:
     * Check if circuit OPEN:
       - If timeout elapsed -> enter HALF_OPEN state
       - Else -> execute fallback_fn immediately if available
     * Call primary_fn with timeout wrapper
     * On success -> _on_success()
     * On exception -> _on_failure(error), try fallback_fn if available
     * Return ResultWithFallback with is_fallback flag
   - _on_success(): Reset failure_count if CLOSED, check success_threshold if HALF_OPEN to CLOSE
   - _on_failure(error): Increment failure_count, trip to OPEN if threshold reached
   - get_state() -> CircuitState: Return current circuit state
   - reset(): Manually reset circuit to CLOSED

Circuit must respect half_open_max_calls before closing circuit.
</description>
