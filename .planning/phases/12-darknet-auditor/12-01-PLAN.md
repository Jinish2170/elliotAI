# Plan 12-01: TOR Client and SOCKS5h Integration

## Context

### Requirement Covered
- **DARKNET-01**: TOR integration with Stem client (adapted to SOCKS5h proxy approach per research)

### Current State (Pre-Plan)

**TOR Integration:**
- No TOR connectivity exists in the codebase
- No SOCKS5 proxy routing for HTTP requests
- All requests use standard HTTP via aiohttp/requests

**Related Infrastructure:**
- aiohttp is already in requirements.txt (>=3.9.0)
- requests is already in requirements.txt
- OSINT orchestrator pattern exists for extending with TOR-aware sources

### Goal State (Post-Plan)

**TOR Client Wrapper:**
- `TORClient` class wrapping aiohttp with SOCKS5h proxy support
- Async context manager for session lifecycle
- Circuit management (connection pooling through aiohttp's built-in pooling)
- DNS resolution on proxy server (socks5h:// for privacy)
- Health check to verify TOR connectivity

**Legal/Privacy Compliance:**
- Read-only OSINT only
- No transaction or purchase capability
- User agent headers set appropriately
- No logging of user-provided .onion URLs
