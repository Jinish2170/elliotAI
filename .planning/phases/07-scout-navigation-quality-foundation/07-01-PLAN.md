# Plan 07-01: Intelligent Scrolling with Lazy-Load Detection

## Requirements

- SCROLL-01: Scout/Vision Agent can scroll pages and capture full screenshot series
- SCROLL-03: Lazy loading detection and handling for complete capture

## Context

**Current State:**
- Scout captures 3 screenshots per page: t0, t+delay, fullpage
- Full-page screenshot uses Playwright's `full_page=True` which scrolls internally
- No intelligent scroll control - page scrolls automatically during full-page capture
- No lazy-load detection - relies on Playwright's default wait behavior

**Problems:**
- Full-page screenshots may miss lazy-loaded content that loads after initial capture
- No visibility into scroll progress or lazy-load timing
- Can't capture screenshots at specific scroll positions for temporal analysis
- No ability to distinguish static vs lazy-loaded content

**Goal State:**
- Intelligent incremental scroll (page height/2 per scroll) with configurable 300-500ms wait
- MutationObserver-based DOM change detection for lazy-load monitoring
- Capture screenshots at scroll intervals with labels indicating scroll position
- Scroll termination: max 15 cycles OR 2 consecutive checkpoints with no new content
- ScrollResult dataclass with scroll state tracking (cycles, stabilization, screenshots captured)

**Key Decisions from CONTEXT.md:**
- Scroll methodology: Intelligent incremental scroll (page height/2 per scroll), not continuous
- Scroll duration: 300-500ms wait after each scroll chunk to allow lazy loading
- Scroll termination: Stop when no new content for 2 consecutive checkpoints OR max 15 cycles
- Scroll detection: Use scroll position vs page height + DOM mutation observer
- Screenshot triggers: Capture screenshot after each scroll segment and upon scroll termination

<description>
Create ScrollOrchestrator class in veritas/agents/scout/scroll_orchestrator.py

Implement ScrollOrchestrator with:
1. Class constants:
   - MAX_SCROLL_CYCLES = 15
   - STABILIZATION_THRESHOLD = 2
   - SCROLL_CHUNK_RATIO = 0.5 (page height/2)
   - SCROLL_WAIT_MS = 400

2. __init__(self, evidence_dir: Path, detector: LazyLoadDetector)

3. scroll_page(page: Page, audit_id: str, screenshot_interval: int=2) -> ScrollResult:
   - Inject LazyLoadDetector at start
   - Loop for max 15 cycles
   - Each iteration:
     * Record current scroll position before scroll
     * Scroll by window.innerHeight * SCROLL_CHUNK_RATIO
     * Wait SCROLL_WAIT_MS (asyncio.sleep)
     * Check for new content via detector.has_new_content()
     * Track cycles_without_content counter
     * Capture screenshot every N cycles (screenshot_interval) or on final cycle
     * Create ScrollState and append to list
   - Return ScrollResult with total_cycles, stabilized, lazy_load_detected, screenshots_captured, scroll_states

4. _capture_scroll_screenshot(page: Page, audit_id: str, cycle: int) -> str:
   - Use filename format: {audit_id}_scroll_{cycle:02d}.jpg
   - Capture full_page=False screenshot
   - Include scroll position in debug log
   - Return filepath string

Must handle errors gracefully and log appropriate debug/warning messages.
