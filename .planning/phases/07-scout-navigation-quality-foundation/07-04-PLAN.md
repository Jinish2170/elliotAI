# Plan 07-04: Quality Foundation - Confidence Scoring & Validation

## Requirements

- QUAL-01: False positive detection criteria with multi-factor validation (2+ sources)
- QUAL-02: Deep statistics and confidence scoring with explainable factors
- QUAL-03: Incremental verification and refinement with state transitions

## Context

**Current State:** After Plan 07-03
- ConsensusEngine framework created with FindingStatus enum, FindingSource, ConsensusResult dataclass
- add_finding() method handles finding aggregation with conflict detection
- _compute_confidence() calculates weighted scores
- _detect_conflict() identifies threat vs safe conflicts

**Remaining Work:**
- ConfidenceScorer for human-readable formatting and tier classification
- ValidationStateMachine for state transition enforcement
- Package exports in __init__.py
- Comprehensive tests covering all quality foundation functionality

**Key Decisions from CONTEXT.md:**
- Scoring factors: Source agreement weight 60%, finding severity 25%, contextual confidence 15%
- Score calculation: 2+ sources high-severity -> 75-100%, 2+ sources medium-severity -> 50-75%, 1 source high-severity -> 40-60%, 1 source medium-severity -> 20-40%
- Visual breakdown: Show score with contributing factors breakdown
- Finding confidence tiers: Confirmed (2+ sources, >=50%), Unconfirmed (1 source, <50%), Pending (insufficient data)
- Conflict detection: If one agent flags safe while another flags threat -> downgrade to "conflicted" status

- [ ] False positive detection with multi-factor validation
