# Plan 07-02: Multi-Page Exploration with Link Discovery

## Requirements

- SCROLL-02: Scout can navigate to multiple pages beyond initial landing page

## Context

**Current State:**
- Scout.investigate() visits only the initial URL provided
- Metadata extraction collects internal_links and external_links but doesn't use them
- navigate_subpage() exists as a lightweight navigation method but only visits one subpage at a time
- No automated discovery or prioritization of navigation links
- No tracking of exploration path or visited URLs

**Problems:**
- Evidence collection limited to landing page only
- No visibility into About, Contact, Privacy pages that often contain important threat indicators
- Manual intervention required to explore additional pages
- No deduplication or filtering of visited URLs
- Can waste time on irrelevant outbound links

**Goal State:**
- Automatic discovery of navigation links from nav bar, footer, and in-page "Learn More" sections
- Priority-based exploration order: nav (1) > footer (2) > content (3)
- Max 8 pages total with timeout per page = baseline 15s + lazy load allowance
- Link deduplication by URL with priority ranking by relevance keywords
- Breadcrumb tracking to avoid revisiting same pages
- ExplorationResult with visited pages, navigation times, and scroll results

**Key Decisions from CONTEXT.md:**
- Pages to visit: Automatically discover About, Contact, Privacy, FAQ, footer links (max 8 pages)
- Link discovery: Prioritize nav bar + footer + in-page "Learn More" sections
- Exploration order: 1) Landing page (already visited), 2) Legal pages (Privacy, Terms), 3) About pages (About, Team), 4) functional pages (Contact, FAQ), 5) outbound footer links
- Navigation method: Follow discovered links, extract all hrefs, deduplicate, prioritize by relevance keywords
- Exploration limit: Timeout per page = baseline 15s + lazy loading wait, max 8 pages total

<description>
Create LinkExplorer class in veritas/agents/scout/link_explorer.py

Implement LinkExplorer with:
1. Class constants:
   - RELEVANT_KEYWORDS: list = ['about', 'contact', 'privacy', 'terms', 'faq', 'team', 'learn more']
   - MAX_PAGES: int = 8

2. __init__(self, base_url: str) - parses base_domain and initializes visited_urls set

3. discover_links(self, page: Page) -> list[LinkInfo]:
   - Call page.evaluate() with JS extraction script that:
     * Finds nav links (nav a, header a, [role="navigation"] a) with priority 1
     * Finds footer links (footer a) with priority 2
     * Finds content links (main a, article a, .content a) matching RELEVANT_KEYWORDS with priority 3
     * Filters to same domain using new URL().hostname check
     * Returns array of objects with url, text, location, priority

   - Filter links via _filter_link() which:
     * Checks domain using _is_same_domain()
     * Skips already visited URLs
     * Skips non-HTTP schemes
     * Adds to visited_urls set

   - Deduplicate by URL using dict
   - Sort unique links by priority (lower = first)
   - Return sorted list of LinkInfo objects

4. _get_link_extraction_script(self) -> str:
   - Returns JavaScript function that extracts and categorizes links
   - Passes RELEVANT_KEYWORDS array as argument to JS
   - Uses new URL() for domain validation

5. _filter_link(self, link_data: dict) -> Optional[LinkInfo]:
   - Returns None for invalid links (no url, wrong domain, already visited, non-HTTP)
   - Returns LinkInfo object for valid links

6. _is_same_domain(self, url_str: str) -> bool:
   - Parses URL and checks if netloc equals base_domain OR ends with '.' + base_domain
   - Handles subdomains correctly (sub.example.com matches example.com)

All methods should have proper type hints and return types.
