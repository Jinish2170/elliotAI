# Plan 07-03: Quality Foundation - Consensus Engine

## Requirements

- QUAL-01: False positive detection criteria with multi-factor validation (2+ sources)
- QUAL-02: Deep statistics and confidence scoring with explainable factors
- QUAL-03: Incremental verification and refinement with state transitions

## Context

**Current State:**
- No multi-factor consensus system
- Findings from Vision, OSINT, Security agents treated independently
- Confidence scoring is per-agency (e.g., SecurityFinding.confidence is 0.0-1.0)
- No conflict detection when agents disagree
- False positive prevention relies on single-agent thresholds

**Problems:**
- Single agent can produce false positives that appear as high-confidence threats
- No mechanism to cross-verify findings across agents
- Users can't tell if a finding is solo-source or multi-source
- Conflicting findings (one agent says safe, another says threat) not detected
- Confidence scores aren't explainable - users can't see how score was calculated

**Goal State:**
- ConsensusEngine with multi-factor validation (2+ sources must agree for "confirmed" status)
- Finding status tiers: CONFIRMED (2+ sources, >=50%), UNCONFIRMED (1 source, <50%), PENDING (insufficient data), CONFLICTED (sources disagree)
- Conflict detection when one agent flags safe while another flags threat
- ConfidenceScorer with explainable scoring breakdown (source agreement 60%, severity 25%, context 15%)
- Visual breakdown showing contributing factors (e.g., "87%: 3 sources agree, high severity")
- ValidationStateMachine for incremental verification and state transitions

**Key Decisions from CONTEXT.md:**
- Source types for consensus: Vision Agent, OSINT Agent, Security Agent - multiple findings from these agents count toward consensus
- Consensus threshold: 2+ distinct agents must corroborate a finding for "confirmed" status (prevents solo-source false positives)
- Partial findings: Single-source findings shown as "unconfirmed" with lower confidence (<50%), displayed separately
- Finding confidence tiers: Confirmed (2+ sources, >=50%), Unconfirmed (1 source, <50%), Pending (insufficient data)
- Conflict detection: If one agent flags safe while another flags threat -> downgrade to "conflicted" status
- Scoring factors: Source agreement weight 60%, finding severity 25%, contextual confidence 15%
- Score calculation: 2+ sources high-severity -> 75-100%, 2+ sources medium-severity -> 50-75%, 1 source high-severity -> 40-60%, 1 source medium-severity -> 20-40%
- Visual breakdown: Show score with contributing factors breakdown
- Dynamic recalculation: Scores update in real-time as additional sources contribute findings

<description>
Create ConsensusEngine class in veritas/quality/consensus_engine.py

Implement ConsensusEngine with:

1. __init__(self, min_sources: int = 2) - stores min_sources threshold and initializes empty findings dict

2. add_finding(self, finding_key: str, agent_type: str, finding_id: str, severity: str, confidence: float, timestamp: Optional[str] = None) -> FindingStatus:
   - Generate ISO timestamp if not provided
   - Create FindingSource instance
   - Get or create ConsensusResult from self.findings[finding_key]
   - Check for conflicts BEFORE adding source via _detect_conflict()
   - If conflict: set status to CONFLICTED, add conflict note, return status
   - Add source to result.sources list
   - Update status based on unique agent count:
     * >= min_sources: CONFIRMED, compute aggregated confidence
     * == 1: UNCONFIRMED, cap confidence at 49.0 (confidence * 80)
     * == 0: PENDING (shouldn't happen)
   - Return updated FindingStatus

3. _detect_conflict(self, existing_sources: list[FindingSource], new_source: FindingSource) -> bool:
   - Maps severity to boolean: CRITICAL/HIGH/MEDIUM/LOW = True (threat), INFO = False (safe)
   - Checks if any source says threat AND any says safe
   - Returns True if conflict detected

4. _compute_confidence(self, result: ConsensusResult) -> float:
   - Calculates weighted confidence (source agreement 60%, severity 25%, context 15%)
   - Severity weights: CRITICAL=1.0, HIGH=0.8, MEDIUM=0.6, LOW=0.4, INFO=0.2
   - Applies CONTEXT.md thresholds:
     * 2+ sources + high severity: capped at 75.0 min
     * 2+ sources + medium severity: 50.0-75.0 range
     * 1 source + high severity: 40.0-60.0 range
     * 1 source + medium severity: 20.0-40.0 range
   - Populates result.confidence_breakdown with source_agreement, severity_factor, context_confidence, source_count
   - Returns score rounded to 1 decimal place

5. get_result(self, finding_key: str) -> Optional[ConsensusResult]

6. get_confirmed_findings(self) -> list[ConsensusResult] - filter for CONFIRMED status

7. get_conflicted_findings(self) -> list[ConsensusResult] - filter for CONFLICTED status

All methods must have proper type hints and return types.
